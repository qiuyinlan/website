# 小说工具网站说明

这个目录目前是一个单文件网页工具，核心文件只有 [正文.html](/home/xuzichun/website/小说工具网站/正文.html)。

目标不是做通用站点，而是快速处理小说生产流程里的几类高频操作：

- 把细纲拆成可直接喂给 AI 的正文提示词
- 从 Gemini 导出的 Markdown 对话里自动提取小说正文
- 把“分卷大纲”拆成多个可复制的章节段提示词

## 入口与结构

- `正文.html`
  - HTML、CSS、JS 都写在同一个文件里
  - 没有构建流程，没有依赖安装步骤
  - 直接用浏览器打开即可

## 当前 3 个标签页

### 1. 细纲转提示词

用途：

- 输入按“第X章”组织的细纲
- 自动拆成多个章节卡片
- 每个卡片可复制一份完整提示词

核心逻辑：

- 按 `第X章` 或 `【第X章` 开头切分
- 输出模板由 `CONFIG.PREFIX` 和 `CONFIG.SUFFIX` 包裹原始章节内容

适合改动的位置：

- `CONFIG.PREFIX`
- `CONFIG.SUFFIX`
- `manualRefresh()` 里 `CONFIG.activeTab === 'gen'` 这一段

### 2. 智能提取正文

用途：

- 输入 Gemini 导出的 Markdown 对话文件
- 只提取 AI 的 `Response`
- 自动识别正文章节并提供复制按钮

核心逻辑：

- 先按 `## Prompt` 分轮次
- 再在每轮里按 `## Response` 只取回答部分
- 在回答里继续按章节标题切分
- 会过滤一些明显噪音，比如：
  - `剧情截断`
  - `核心要求`
  - `Exported:`
  - `Link:`
  - `Powered by`

正文识别规则：

- 标题支持这几类开头：
  - `1.`
  - Markdown 标题里的数字段落
  - `第X章`
- 正文长度目前要求大于 `100` 字才会收录

适合改动的位置：

- `noiseKeywords`
- `blocks` 的切分正则
- `if (body.length > 100)` 这类阈值

### 3. 分卷大纲提取

用途：

- 输入整份“分卷大纲”
- 自动按 `第1卷 / 第2卷 / ...` 拆卷
- 每一卷生成 6 份可直接喂给 Gemini 的提示词，并按卷号自动顺延章节区间

章节区间规则：

- 第1卷：`1-60`
- 第2卷：`61-120`
- 第3卷：`121-180`
- 依此类推，每卷继续往后顺延 60 章

复制出去的内容结构：

1. 先放这一卷的完整大纲
2. 再拼接“创作分章节大纲”的固定提示词模板
3. 自动把卷号和章节范围填进去

相关函数：

- `CONFIG.VOLUME_SEGMENT_SIZE`
  - 每个提示词块的章节跨度，默认 `10`
- `CONFIG.VOLUME_SEGMENT_COUNT`
  - 每卷生成几个提示词块，默认 `6`
- `getVolumeSegments(volumeTitle)`
  - 根据卷号自动算出本卷对应的 6 段章节范围
- `getVolumeNumber(title)`
  - 从 `第X卷` 标题里提取卷号
- `buildVolumePrompt(volume, segment)`
  - 负责拼接最终复制给 Gemini 的完整提示词

如果下个 AI 要改这里，常见需求通常有三类：

- 改章节区间
  - 直接改 `CONFIG.VOLUME_SEGMENT_SIZE` 或 `CONFIG.VOLUME_SEGMENT_COUNT`
- 改提示词模板文案
  - 直接改 `buildVolumePrompt()`
- 改分卷识别规则
  - 改 `CONFIG.activeTab === 'volumes'` 分支里的切分正则

## 状态持久化

页面使用浏览器 `localStorage` 保存每个标签页的输入内容。

当前 key：

- `novel-pipeline-clean-v9`

保存结构：

```js
{
  gen: '',
  parse: '',
  volumes: ''
}
```

行为说明：

- 切换标签页时，会记住当前输入框内容
- 刷新页面后会恢复
- 只有点击“清空”才会清掉当前标签页的数据

## 交互要点

- 左侧栏显示当前模式下拆出的章节或分卷列表
- 右侧卡片提供复制按钮
- 拖入文件时，会把文件文本直接读进输入框
- 输入内容小于 `50000` 字时会自动刷新解析

## 最常见的后续改法

- 把 6 个固定章节段改成可输入
- 给“分卷大纲提取”增加“复制全部 6 份”
- 支持 `第01卷`、中文数字卷号、更多格式的大纲标题
- 给不同模式单独写说明文字，避免用户混淆输入格式
